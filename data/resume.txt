Resume Guidelines for Placement (1 Page Preferred)

Golden Rules
- Keep it one page for fresher roles.
- Use strong action verbs: built, implemented, optimized, designed.
- Add measurable impact: reduced latency by 20%, improved accuracy by 8%.
- Tailor keywords to role description.
- Avoid generic lines like "hardworking" without evidence.

Section Order (Recommended)
1. Name + contact + links (GitHub, LinkedIn, portfolio)
2. Education
3. Skills
4. Projects
5. Experience / Internship
6. Achievements / Certifications

Project Bullet Formula
Action + Tech + Problem + Impact
Example:
- Built a Streamlit-based placement prep assistant using RAG (FAISS + sentence-transformers) and Groq LLM API, reducing answer latency for common queries by ~30% through cached indexing.

Common Resume Mistakes
- Too many tools listed without proof in projects.
- No metrics in project/experience bullets.
- Poor formatting and inconsistent tense.
- Writing paragraphs instead of concise bullets.
